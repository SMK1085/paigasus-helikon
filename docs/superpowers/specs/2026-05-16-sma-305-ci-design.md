# SMA-305 — CI: build, test, clippy, fmt, docs, doc-coverage

**Linear issue**: [SMA-305](https://linear.app/smaschek/issue/SMA-305/ci-build-test-clippy-fmt-matrix)
**Status**: design approved 2026-05-16
**Branch**: `feature/sma-305-ci-build-test-clippy-fmt-matrix`
**Depends on**: SMA-304 (workspace skeleton — landed)
**Related**: SMA-309 (branch protection wires the required-status-check names this PR produces)

## 1. Goal & non-goals

**Goal.** Replace the placeholder `build` job in `.github/workflows/ci.yml` with the full quality gate the project needs to merge real code: `fmt`, `clippy`, `test` (matrix), `docs`, `doc-coverage`, plus a separate `msrv.yml` that verifies the declared `rust-version` is truthful. Total wall-clock under 8 minutes on a no-op PR. Workspace-wide docstring coverage gated at ≥ 80%, configurable via env, with per-crate breakdown in the PR Checks tab.

**Non-goals.**
- Branch protection itself (SMA-309).
- Conventional-Commits PR-title check (SMA-335).
- `cargo-deny` / `audit` jobs (separate tickets; SMA-309 lists them as required so they will be added before branch protection flips on).
- Release-plz integration.
- Writing actual docstrings for crates that don't yet have public API. The bootstrap PR establishes the *baseline* coverage number; subsequent tickets ratchet it.

## 2. Workflow files

```text
.github/
└── workflows/
    ├── ci.yml      (rewritten — fmt, clippy, test matrix, docs, doc-coverage)
    └── msrv.yml    (new — cargo-msrv verify)
scripts/
└── check-doc-coverage.sh   (new — bash + jq aggregator)
CONTRIBUTING.md             (new — policies)
```

The existing minimal `ci.yml` is replaced wholesale, not extended in place — the bootstrap job has no signal worth preserving.

## 3. Job matrix — pruning rationale

A full `{3 OS × 2 toolchains × 5 jobs}` cross product is 30 invocations and overshoots the 8-min budget on Windows runners (which are ~2× slower than Linux for cold cargo builds). The pruned matrix:

| Job ID | OS | Toolchain | Notes |
|---|---|---|---|
| `fmt` | ubuntu-latest | stable | Pure formatting — platform/version invariant |
| `clippy` | ubuntu-latest | stable | Lints are toolchain-coupled; running on MSRV would emit noise from lints that don't exist on 1.75 |
| `test` | ubuntu/macos/windows × stable/1.75 | matrix(6) | Only job that exercises platform-conditional code paths |
| `docs` | ubuntu-latest | stable | Rustdoc output is platform-invariant |
| `doc-coverage` | ubuntu-latest | nightly (pinned) | Requires `-Z unstable-options` |

**Required-status-checks (consumed by SMA-309):**
- `ci / fmt`
- `ci / clippy`
- `ci / test (ubuntu-latest, stable)` — the canonical green-light test; the macOS/Windows/MSRV variants run as additional signal but do not block merge
- `ci / docs`
- `ci / doc-coverage`

`msrv / verify` is **not** in SMA-309's required-check list. It runs as a signal so MSRV truthfulness drift is visible in PRs without becoming a merge gate.

## 4. `ci.yml` — full file

```yaml
name: ci

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name == 'pull_request' }}

env:
  CARGO_TERM_COLOR: always
  NIGHTLY_TOOLCHAIN: nightly-2026-05-01

jobs:
  fmt:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt
      - run: cargo fmt --all -- --check

  clippy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: clippy
      - uses: Swatinem/rust-cache@v2
      - run: cargo clippy --workspace --all-features --all-targets -- -D warnings

  test:
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        toolchain: [stable, "1.75"]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@master
        with:
          toolchain: ${{ matrix.toolchain }}
      - uses: Swatinem/rust-cache@v2
      - run: cargo test --workspace --all-features

  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: Swatinem/rust-cache@v2
      - run: cargo doc --workspace --all-features --no-deps
        env:
          RUSTDOCFLAGS: "-D warnings"

  doc-coverage:
    runs-on: ubuntu-latest
    env:
      DOC_COVERAGE_THRESHOLD: "80"
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@master
        with:
          toolchain: ${{ env.NIGHTLY_TOOLCHAIN }}
      - uses: Swatinem/rust-cache@v2
      - run: bash scripts/check-doc-coverage.sh
        env:
          NIGHTLY_CHANNEL: ${{ env.NIGHTLY_TOOLCHAIN }}
```

**Note on `fail-fast: false`** for `test`: when MSRV breaks because a transitive dep raised its floor, we want to see whether stable still passes, not have the whole matrix abort on first failure.

**Note on warning-as-error scope.** Lint enforcement is opt-in per job: `clippy` uses `-- -D warnings`, `docs` sets `RUSTDOCFLAGS=-D warnings`. Workflow-level `RUSTFLAGS=-D warnings` is deliberately *not* set, because it would scope into the `doc-coverage` job's nightly rustc compile and could fail on nightly-only lints (`stable_features`, future-compat) that have nothing to do with the coverage signal we want.

## 5. `msrv.yml` — full file

```yaml
name: msrv

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: msrv-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name == 'pull_request' }}

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: Swatinem/rust-cache@v2
      - uses: taiki-e/install-action@v2
        with:
          tool: cargo-msrv
      # cargo-msrv has no --workspace flag; verify one representative crate
      # whose rust-version inherits from [workspace.package].
      - run: cargo msrv --path crates/paigasus-helikon-core verify
```

`taiki-e/install-action` fetches a prebuilt `cargo-msrv` binary in seconds; `cargo install cargo-msrv` would add a minute of compile time per run.

`cargo msrv verify` reads `rust-version` from each crate's `Cargo.toml` (which inherits from `[workspace.package]`) and confirms the workspace actually builds on that version. It's the inverse complement of the `test` matrix's MSRV row: the matrix asks "does MSRV still work today?" and `cargo-msrv verify` asks "is the declared MSRV truthful, or are we silently relying on a newer feature?"

## 6. `scripts/check-doc-coverage.sh` — design

```bash
#!/usr/bin/env bash
# Requires bash >= 4.0 (mapfile). On macOS: `brew install bash`.
set -euo pipefail

THRESHOLD="${DOC_COVERAGE_THRESHOLD:-80}"
NIGHTLY="${NIGHTLY_CHANNEL:-nightly}"

# Crates excluded from the aggregated gate. The CLI is allowed to omit docs
# until its public surface stabilizes (SMA-305 §7), so it should not contribute
# to the workspace denominator either.
EXCLUDED_CRATES=("paigasus-helikon-cli")

is_excluded() {
  local needle="$1"
  for c in "${EXCLUDED_CRATES[@]}"; do
    [[ "$c" == "$needle" ]] && return 0
  done
  return 1
}

# Run cargo metadata in the main shell so set -e applies; using a
# herestring keeps jq's exit status visible too. Process substitution
# `<(...)` runs in a subshell where pipefail does NOT propagate, so a
# failure of cargo metadata or jq would silently produce an empty list
# and the script would report a vacuous 100% pass.
metadata="$(cargo metadata --format-version 1 --no-deps)"
mapfile -t crates < <(jq -r '.packages[] | .name' <<< "$metadata")

json="$(mktemp)"
trap 'rm -f "$json"' EXIT

total_items=0
total_documented=0
summary_rows=()
failed_crates=()

for crate in "${crates[@]}"; do
  if is_excluded "$crate"; then
    summary_rows+=("| \`${crate}\` | _excluded_ | _excluded_ | _opt-out (§7)_ |")
    continue
  fi
  if ! cargo "+${NIGHTLY}" rustdoc -p "$crate" --all-features -- \
    -Z unstable-options --show-coverage --output-format json \
    > "$json" 2> /dev/null; then
      echo "::warning::rustdoc --show-coverage failed for ${crate}; marking crate as failed"
      failed_crates+=("$crate")
      summary_rows+=("| \`${crate}\` | n/a | n/a | _rustdoc error_ |")
      continue
    fi

  # JSON shape: { "<file>": { "total": N, "with_docs": M, ... }, ... }
  crate_total=$(jq '[.[].total] | add // 0' "$json")
  crate_docs=$(jq  '[.[].with_docs] | add // 0' "$json")
  total_items=$((total_items + crate_total))
  total_documented=$((total_documented + crate_docs))

  if [[ "$crate_total" -eq 0 ]]; then
    pct="n/a"
  else
    pct=$(awk "BEGIN { printf \"%.1f\", ($crate_docs / $crate_total) * 100 }")
  fi
  summary_rows+=("| \`${crate}\` | ${crate_total} | ${crate_docs} | ${pct}% |")
done

if [[ "$total_items" -eq 0 ]]; then
  workspace_pct="100.0"   # vacuous truth — no public items to document yet
  note="_No public items in workspace yet — baseline pass._"
else
  workspace_pct=$(awk "BEGIN { printf \"%.1f\", ($total_documented / $total_items) * 100 }")
  note=""
fi

# Emit GitHub step summary
{
  echo "## Doc coverage"
  echo
  echo "Workspace: **${workspace_pct}%** (${total_documented}/${total_items}) — threshold: **${THRESHOLD}%**"
  [[ -n "$note" ]] && { echo; echo "$note"; }
  echo
  echo "| Crate | Total | Documented | Coverage |"
  echo "| --- | ---: | ---: | ---: |"
  printf '%s\n' "${summary_rows[@]}"
} >> "${GITHUB_STEP_SUMMARY:-/dev/stdout}"

if [[ "${#failed_crates[@]}" -gt 0 ]]; then
  echo "::error::Doc coverage could not be computed for: ${failed_crates[*]}"
  exit 1
fi

# Gate
awk "BEGIN { exit !($workspace_pct >= $THRESHOLD) }" || {
  echo "::error::Doc coverage ${workspace_pct}% is below threshold ${THRESHOLD}%"
  exit 1
}
```

**Design notes:**
- Per-crate rustdoc invocation (not `--workspace`): `--show-coverage` only emits the coverage block for the crate currently being processed; running it for the whole workspace at once silently reports just the root crate.
- **CLI excluded from aggregation.** The `EXCLUDED_CRATES` list mirrors the §7 `missing_docs = "allow"` posture: if a crate is allowed to skip docs, it should not contribute to the workspace denominator. Otherwise the "allow" only silences the lint while the CLI's empty `fn main` still drags the percentage. Excluded crates still appear in the step summary marked `_opt-out_` so the exception stays visible to reviewers.
- `total == 0 → 100%`: the bootstrap PR has stub crates with zero `pub` items. Without this branch the gate would either divide by zero or fail at 0%.
- `awk` rather than `bc` for the percentage comparison — `bc` isn't installed on GitHub's Ubuntu runners by default; `awk` always is.
- Falls back to `/dev/stdout` when `GITHUB_STEP_SUMMARY` is unset (i.e. running locally).
- The local invocation is `DOC_COVERAGE_THRESHOLD=80 scripts/check-doc-coverage.sh` after installing the pinned nightly (`rustup toolchain install nightly-2026-05-01`).

## 7. Workspace `missing_docs` enforcement — `[workspace.lints]`

Instead of `#![warn(missing_docs)]` in every crate root (and `#![cfg_attr(not(doc), allow(missing_docs))]` on the CLI), declare the policy once in the root `Cargo.toml`:

```toml
# Cargo.toml (workspace root) — append to existing file
[workspace.lints.rust]
missing_docs = "warn"
```

Every non-CLI crate adds one line to its `Cargo.toml`:

```toml
[lints]
workspace = true
```

The CLI crate (`paigasus-helikon-cli`) overrides locally per the ticket's "initially permitted" clause:

```toml
[lints.rust]
missing_docs = "allow"  # SMA-305: opt out until Stage 3 public API stabilizes
```

This requires Cargo 1.74+. MSRV is 1.75, so the feature is available across the whole supported toolchain range. The `[workspace.lints]` table is **inherited via opt-in** (`[lints] workspace = true`) — crates without that table get no lint override, which matches the existing workspace-inheritance pattern from SMA-304.

**Why not `#![warn(missing_docs)]` per crate?**
- 13 crate roots to keep in sync vs. one workspace declaration.
- `#![cfg_attr(not(doc), allow(missing_docs))]` (from the ticket) is brittle: `not(doc)` is true during `cargo build`/`cargo test` but false during `cargo doc`, meaning warnings only appear during the docs job. A `[lints] workspace = true` opt-in plus per-crate `missing_docs = "allow"` makes the policy auditable in `Cargo.toml` instead of buried in source.
- Deviates from the ticket's literal wording; recorded in §10.

## 8. `CONTRIBUTING.md` — content outline

New file, target ~60 lines, structured as:

1. **Branch naming.** Exact regex from SMA-309 (`^(feature|hotfix)\/[a-z0-9._-]+$`) and a note that Linear's "Copy git branch name" produces compliant names.
2. **Commit messages.** `<type>(<scope>): SMA-### <message>` format, examples for `feat`, `fix`, `docs`, `ci`, `chore`, `refactor`. Pointer to SMA-335 when the PR-title gate lands.
3. **MSRV.** Statement of `1.75`. Policy: if a dependency raises the floor, bump `rust-version`; never downgrade the dependency.
4. **Docstring coverage.** ≥ 80% workspace-wide, gated in CI. Local check: `DOC_COVERAGE_THRESHOLD=80 scripts/check-doc-coverage.sh` (after `rustup toolchain install nightly-2026-05-01`). Pointer to the `[workspace.lints]` config so contributors know where the policy lives.
5. **Local pre-PR checklist.** The five commands from §9.
6. **CI gates merge.** One sentence: CI failures block merge; don't ask reviewers to relitigate what the linter already said.

No code-of-conduct, no DCO, no contributor agreement — out of scope.

## 9. Verification (post-implementation)

These must all pass locally on the feature branch before requesting review:

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-features --all-targets -- -D warnings
cargo test --workspace --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --workspace --all-features --no-deps
rustup toolchain install nightly-2026-05-01      # one-time
DOC_COVERAGE_THRESHOLD=80 scripts/check-doc-coverage.sh
cargo msrv --path crates/paigasus-helikon-core verify  # one-time: cargo install cargo-msrv (or use binstall)
```

CI itself must run green on the SMA-305 PR (and stay under 8 min wall-clock on the no-op PR after this one) before merge.

## 10. Risks & deviations from the ticket

1. **`[workspace.lints]` instead of `#![warn(missing_docs)]` per crate.** The ticket prescribes the per-crate attribute. We use the workspace lint table because it centralizes the policy, requires one line per crate, and avoids the `#![cfg_attr(not(doc), allow(missing_docs))]` brittleness (where lints only fire under `cargo doc`). Mechanism differs; the gate is identical.
2. **Pruned matrix for fmt/clippy/docs.** The ticket's wording ("Matrix: ubuntu-latest, macos-latest, windows-latest × stable, MSRV (1.75)") could be read as applying the matrix to every job. We apply it only to `test`. fmt and clippy lints are toolchain-coupled (clippy on MSRV emits lints that don't exist on stable, generating noise); docs output is platform-invariant. Net: 10 jobs instead of 30, well inside the 8-min budget.
3. **Nightly pinned to `nightly-2026-05-01`.** The ticket says "nightly toolchain required." Floating `nightly` is a CI footgun — the rustdoc JSON schema is `-Z unstable-options` and can change between nightlies. A date pin localizes the bump to a single `NIGHTLY_TOOLCHAIN:` line in `ci.yml`. Bumping is a follow-up chore, not an emergency.
4. **MSRV verification both in `ci.yml` matrix and `msrv.yml`.** Two complementary checks: the matrix asks "does it compile today on 1.75?" and `cargo msrv verify` asks "is the declared MSRV the actual minimum?" Either alone misses a class of regression. The runtime cost is small (`cargo-msrv verify` is a single compile, not a binary search).
5. **`total == 0 → 100%` in `check-doc-coverage.sh`.** Required because the bootstrap PR baseline has near-zero public items; a strict-percentage gate would either divide by zero or fail at 0%. Documented as a script-level invariant so the first ticket that adds public API isn't surprised.
6. **CLI excluded from doc-coverage aggregation.** The ticket's `missing_docs = "allow"` exemption for the CLI only silences the lint; without excluding it from the aggregator the CLI's `fn main` (no docstring) still counts against the workspace total. We exclude it via `EXCLUDED_CRATES` in the script so the exemption is consistent. Exclusion is visible in the step-summary table (`_opt-out_`).
7. **`CONTRIBUTING.md` is minimal.** Branch naming + commit format + MSRV + doc-coverage policy only. No CoC/DCO/CLA scaffolding. Add when the project grows external contributors.
8. **MSRV `test` row may fail on first run.** `opentelemetry 0.27` or `rmcp 0.16+` may demand >1.75. Per CLAUDE.md and the SMA-304 spec, the remedy is to bump `rust-version` to what `cargo` demands, not to downgrade the dep. Surface the failure; bump in a follow-up commit on the same PR.
9. **`ci / commits`, `pr-title`, `cargo-deny`, `audit`.** Listed by SMA-309 as required status checks but produced by separate tickets. Out of scope for SMA-305; SMA-309 will be implementable only after those tickets land.
